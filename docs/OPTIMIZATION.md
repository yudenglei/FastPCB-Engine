# FastPCB-Engine 完整优化列表

## 1. 内存优化核心

### 1.1 参数值紧凑存储 (ParamValue Union)
- **问题**: 每个坐标/尺寸参数创建OCAF Label，内存占用约100字节
- **方案**: 使用union结构，16字节存储
- **效果**: 节省 **84%** 内存

```cpp
struct ParamValue {
    union {
        double doubleValue_;      // 8字节
        uint32_t varPoolIndex_; // 4字节
    };
    uint16_t flags_;             // 2字节
};
```

### 1.2 预分配内存池
- **问题**: 频繁new/delete导致内存碎片
- **方案**: 启动时预分配大块内存
- **效果**: 消除碎片，减少系统调用

### 1.3 Reference池化
- **问题**: OCAF Reference创建额外Label
- **方案**: 整数ID代替Reference + 字符串池化
- **效果**: 节省 **80%+** 内存

### 1.4 延迟加载
- **问题**: 启动时加载所有数据
- **方案**: 按需实例化Shape
- **效果**: 降低内存峰值

## 2. 参数化依赖图（有向图）

### 2.1 DAG结构
- **问题**: 参数化变量有依赖关系，需要拓扑排序
- **方案**: 有向无环图（DAG）管理
- **效果**: 自动检测循环依赖，拓扑排序

### 2.2 增量更新
- **问题**: 参数变化需要更新所有下游节点
- **方案**: 记录脏标志，只更新受影响节点
- **效果**: O(n) → O(节点出度)

### 2.3 循环检测
- **问题**: 防止无限递归
- **方案**: DFS检测循环依赖
- **效果**: 提前预警

## 3. 数据结构优化

### 3.1 OCAF Label预分配策略
```
Label 1-100:   系统保留
Label 101-1000: 变量池 (前置!)
Label 1001-2000: 层叠结构
Label 2001-100000: 器件
Label 100001-200000: 网络
Label 200001-1000000: 几何数据
```

### 3.2 分层数据结构
- Via: PadTop + PadBottom + Drill
- BondWire: 端点 + 曲线 + 曲面
- Trace: 路径 + 线宽 + 层
- Surface: 边界 + 挖孔 + 网络

## 4. 性能优化

### 4.1 批量操作
- 参数批量设置 (setValuesBatch)
- 批量创建器件 (createBatch)
- 批量刷新 (refreshBatch)

### 4.2 多线程加载
- 并行加载不同分区
- 线程安全的对象池
- 8线程并行处理

### 4.3 LRU缓存
- Shape缓存: 1024个
- 属性缓存: 2048个

### 4.4 增量更新
- 参数变化只更新受影响区域
- 后台线程重算
- 脏标志追踪
