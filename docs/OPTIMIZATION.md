# FastPCB-Engine 性能优化总结

## 优化列表 (2026-03-19)

### 1. 内存优化核心

#### 1.1 参数值紧凑存储 (ParamValue Union)
- **问题**: 每个坐标/尺寸参数创建OCAF Label，内存占用约100字节
- **方案**: 使用union结构，16字节存储
  - 未设置变量: 直接存储double值 (8字节)
  - 设置变量: 存储变量池索引 (4字节) + 标志 (2字节)
- **效果**: 节省 **84%** 内存

```cpp
struct ParamValue {
    union {
        double doubleValue_;      // 8字节
        uint32_t varPoolIndex_; // 4字节
    };
    uint16_t flags_;             // 2字节
    // 总计: 16字节 (对齐)
};
```

#### 1.2 预分配内存池
- **问题**: 频繁new/delete导致内存碎片和开销
- **方案**: 启动时预分配大块内存
  - 参数池: 100万个槽位
  - 几何池: 50万个对象
  - 字符串池: 64MB
- **效果**: 消除碎片，减少系统调用

#### 1.3 Reference池化
- **问题**: 每个OCAF Reference创建额外Label，1000万Reference = 500MB
- **方案**: 
  - 使用整数ID代替Reference
  - 字符串池化：相同名称只存储一份
  - 惰性解析：仅在需要时解析
- **效果**: 节省 **80%+** 内存

#### 1.4 延迟加载
- **问题**: 启动时加载所有数据，内存峰值高
- **方案**:
  - 扩展数据延迟加载
  - Shape几何延迟实例化
  - 按需建立索引

### 2. 性能优化

#### 2.1 批量操作
- 参数批量设置
- 批量创建器件
- 批量刷新

#### 2.2 多线程加载
- 并行加载不同分区
- 线程安全的对象池

#### 2.3 LRU缓存
- Shape缓存: 1024个
- 属性缓存: 2048个
- 自动淘汰冷数据

#### 2.4 增量更新
- 参数变化只更新受影响区域
- 后台线程重算

### 3. 数据结构优化

#### 3.1 OCAF Label预分配策略
```
Label 1-100:   系统保留
Label 101-1000: 变量池 (前置!)
Label 1001-2000: 层叠结构
Label 2001-100000: 器件
Label 100001-200000: 网络
Label 200001-1000000: 几何数据
```

#### 3.2 分层数据结构
- Via: PadTop + PadBottom + Drill
- BondWire: 端点 + 曲线 + 曲面
- Trace: 路径 + 线宽 + 层
- Surface: 边界 + 挖孔 + 网络

#### 3.3 参数到Shape联动
- Observer模式
- 参数变化自动刷新
- 脏标志追踪

### 6. 关键技术点

1. **Union存储**: 8字节vs100字节
2. **内存池**: 预分配+对象复用
3. **整数ID**: 代替Label引用
4. **延迟加载**: 按需实例化
5. **批量操作**: 减少系统调用
6. **Observer模式**: 参数-Shape联动
